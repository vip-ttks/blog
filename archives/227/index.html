nginx配置代理，以及缓存，删除缓存

工作技术 admin 6个月前 (08-28) 46℃

项目需要用到，Nginx需要配置代理到php服务器，然后需要按照标准Http进行来缓存，然后提供对外接口可以用来删除缓存

其中http缓存参数是Expires 或者Cache-Control: max-age具体原理自行百度

下面是一个可用的Nginx配置文件

user  root;
worker_processes  1;
worker_rlimit_nofile 102400;
events {
    worker_connections  102400;
    use epoll;
}
http {
    include       mime.types;
    default_type  application/octet-stream;
    proxy_cache_path /tmp/cache levels=1:2 keys_zone=cache_one:100m max_size=1g inactive=60m use_temp_path=off;
    server {
        listen 8085;
        server_name  ~^\d+\.\d+\.\d+\.\d+$;     
        #resolver 114.114.114.114 8.8.8.8 valid=30s;              #动态DNS解析
        
        location ~ /purge(/.*) {
               #allow              127.0.0.1;
               #allow             192.168.2.112;
               #deny               all;
               allow all;
               proxy_cache_purge  cache_one $1$is_args$args;
        }
        
        location / {
                        #首页也要缓存代理
                        rewrite_by_lua '
                        if ngx.var.uri==nil or string.len(ngx.var.uri)==1 then
                        if ngx.var.args~=nil then
                        ngx.exec("/index.html?"..ngx.var.args);
                        else
                        ngx.exec("/index.html")
                        end
                        end
                ';
                proxy_cache cache_one;
                proxy_connect_timeout 60s;
                proxy_read_timeout 5400s;
                proxy_send_timeout 5400s;
                #proxy_cache_key "$host:$server_port$uri$is_args$args";
                proxy_cache_key "$uri$is_args$args";
                proxy_cache_valid 200 302 10m;
                #proxy_set_header Host $host; 
                proxy_set_header Host "nb-t.2r3u48np.net";
                proxy_set_header X-Real-IP $remote_addr;  
                proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
                proxy_pass http://nb-t.2r3u48np.net:80/;
                add_header X-Cache $upstream_cache_status;
        }
    }
}

如果Nginx所在的电脑IP是192.168.4.164，那么访问

http://192.168.4.164:8085/clb/index.php?c=datav2&m=main&no=1

就会自动代理到

http://nb-t.2r3u48np.net:80/clb/index.php?c=datav2&m=main&no=1

如果有缓存的话也会使用缓存

想要删除该链接的缓存就访问http://192.168.4.164:8085/purge/clb/index.php?c=datav2&m=main&no=1

如果要强制缓存的话 就在location语句里面添加如下

proxy_ignore_headers X-Accel-Expires Expires Cache-Control Set-Cookie;
