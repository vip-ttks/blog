C++中放弃使用cout只使用printf

工作技术 admin 7个月前 (07-18) 12℃

cout被大家认为简洁，安全，但是效率真心是无法忍受的，下面的测试代码

#include <stdio.h>
#include <stdlib.h>
#include <windows.h>
#include <iostream>
using namespace std;
int _tmain(int argc, _TCHAR* argv[])
{
	DWORD t,t2;
	{
		t=::GetTickCount();
		for(int i=0;i<10000;i++)
		{
			printf("i=%d\r\n",i);
		}
		t=::GetTickCount()-t;
	}
	ios::sync_with_stdio(false);
	{
		t2=::GetTickCount();
		for(int i=0;i<10000;i++)
		{
			cout<<"i="<<i<<endl;
		}
		t2=::GetTickCount()-t2;
	}
	printf("t=%d t2=%d\r\n",t,t2);
	return 0;
}

编译环境是VS2005，先编译debug版本结果是t=998 t2=4603

再换成release版本 结果是t=920 t2=4555

所以说不用cout的第一个理由就是效率问题
